## Cómo usar esta documentación

Cada módulo es un archivo Markdown independiente. Podés:

- Leerlos en orden para seguir la currícula completa.
- Ir directo al módulo que necesitás como referencia rápida.
- Usar los comandos de la sección de **Troubleshooting** en tu terminal.

---

## Prerequisitos

No se requiere software especial para leer la documentación. Para ejecutar los comandos del Módulo 5:

- Linux / macOS: disponibles nativamente en la terminal.
- Windows: usar **WSL2**, **Git Bash** o **PowerShell** (algunos comandos varían).
